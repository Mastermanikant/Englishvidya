# Module 12: उपयोग ट्रैकिंग, मल्टी‑डिवाइस, विज्ञापन, रेफ़रल और कम्युनिटी रणनीति - English Vidya

## 1. उपयोग‑ट्रैकिंग और ऑटो‑स्क्रीन‑ऑफ़ (Usage Tracking & Auto‑Screen‑Off)
| उद्देश्य | विवरण |
|---|---|
| **सेशन टाइम‑ट्रैकिंग** | ब्राउज़र में `Performance.now()` या `Date.now()` द्वारा प्रत्येक पेज‑सेशन की अवधि मापें और **sessionStorage** में रखें। जब उपयोगकर्ता टैब बंद करे, पेज नेविगेट करे या `visibilitychange` इवेंट आये, तो कुल समय को एक **एकल डेटाबेस‑राइट** के रूप में Cloudflare Worker पर भेजें (`/api/usage`). यह हर 24 घंटे में **23:59:59** पर दैनिक संयुक्त रिपोर्ट बनाता है। |
| **ऑटो‑स्क्रीन‑ऑफ़** | मोबाइल पर `setTimeout` (60 सेकंड) के बाद `document.body.style.opacity = 0.3` और एक ओवरले "स्क्रीन स्लीप मोड" दिखाएँ। उपयोगकर्ता टच करने से स्क्रीन फिर से ऑन हो जाएगी। यह डिवाइस‑लेवल पर क्लाइंट‑साइड काम करता है, सर्वर को कोई लोड नहीं देता। |
| **डेटा मॉडल (D1)** | ```sql
CREATE TABLE usage_log (
  user_id TEXT NOT NULL,
  device_id TEXT NOT NULL,
  date TEXT NOT NULL,          -- YYYY‑MM‑DD
  total_seconds INTEGER NOT NULL,
  sessions INTEGER NOT NULL,
  PRIMARY KEY(user_id, device_id, date)
);
``` |
| **रिपोर्ट** | हर रात 23:59:59 पर एक Cloudflare Worker (`/api/daily_summary`) सभी उपयोग‑डेटा को इकट्ठा कर एक **डैशबोर्ड** JSON बनाता है (पिछले 30 दिन)। यह UI में “आपने आज 12 मिनट पढ़ा, चलिए 15 मिनट तक पहुँचें” जैसा संकेत दिखाएगा। |

## 2. मल्टी‑डिवाइस लॉग‑इन (Multi‑Device Login)
* **डिवाइस‑आईडी:** जब उपयोगकर्ता पहली बार साइन‑इन करे, तब एक **UUID** को `localStorage` में स्टोर करें → `device_id`।
* **सत्र‑सीमा:** JWT में `device_id` और `exp` रखें। हर रीक्वेस्ट पर Worker जाँचता है कि उसी `user_id` के साथ अधिकतम **3** सक्रिय `device_id`s हों (कस्टम सेटिंग)। यदि नई डिवाइस जुड़ती है, तो सबसे पुरानी (आखिरी उपयोग) डिवाइस स्वचालित रूप से निरस्ती (revoke) की जा सकती है।
* **सुरक्षा‑जुगाड़:** डिवाइस‑ऑन‑बोर्ड फ़िंगरप्रिंट लाइब्रेरी (जैसे `fingerprintjs`) को जोड़ें ताकि बॉट‑खाता बनाना कठिन हो।

## 4. उपयोग‑ड्रॉप‑फ़ीडबैक और सुधार (Drop‑out Feedback)
1. **ड्रॉप‑डिटेक्शन:** जब उपयोगकर्ता **सेशन‑टाइमआउट** या **स्क्रीन‑ऑफ़** के बाद 2 मिनट तक पुनः लॉग‑इन नहीं करता, तो 24 घंटे बाद एक पुश‑नोटिफ़िकेशन/ई‑मेल भेजें – “क्या पढ़ाई में कोई दिक्कत आ रही है? हमें बताइए।”
2. **सर्वे‑फ़ॉर्म:** छोटा फॉर्म (गूगल फ़ॉर्म या Typeform) जो “कन्टेंट कठिन/बहुत आसान” पूछता है। उत्तर सीधे Cloudflare‑KV में स्टोर होंगी। |

## 5. प्रारम्भिक मोनेटाइज़ेशन योजना (Initial Monetisation)
| मॉडल | विवरण |
|---|---|
| **फ्री‑टियर + 30‑सेक‑एड (5‑घंटे ad‑free)** | उपयोगकर्ता 30‑सेकण्ड का वीडियो‑एड देखेगा → 5 घंटे बिना विज्ञापन का अनुभव। इस समय में वह निरंतर पढ़ सकता है। |
| **बैनर‑एड‑फ्री (जब‑एड‑देखा‑जा‑ता‑है)** | यदि उपयोगकर्ता एड देखता है, तो उसी सत्र में सभी बैनर‑एड हटे। |
| **वॉचर‑एड‑इन (Watch‑Ad‑Earn)** | हर 10 मिनट पढ़ाई पर “Earn ₹0.5” का ऑफ़र, विज्ञापन देख कर। कमाई को **रिफ़रल‑कोड** के साथ शेयर कर सके। |
| **डेली रिवॉर्ड (Daily Bonus)** | लगातार 7 दिन पढ़ने पर “Free PDF Chapter” या “Extra 5 मिनट एड‑फ्री” प्राप्त। |
| **रेफ़रल‑प्रोग्राम** | प्रत्येक सफल रेफ़रल (नया उपयोगकर्ता 30 मिनट सक्रिय) पर **₹1** (या पॉइंट) देता है, लेकिन **फ़्रॉड‑प्रिवेंशन**: रेफ़रल कोड केवल **डिवाइस‑लेवल** (एक‑डिवाइस‑पर‑कोड) हो, और रेफ़रल की **समान‑IP** या **समान‑फ़िंगरप्रिंट** पर प्रतिबंध। |
| **एंटी‑बॉट/फ्रॉड** | reCAPTCHA v3 (Google) एम्बेड, और **डिवाइस‑फिंगरप्रिंट** सीमित संख्या। प्रत्येक दिन 5 से कम रेफ़रल का उपयोग मान्य, अन्य को ब्लॉक। |

## 6. कम्युनिटी (Community) एवं रोल‑बेस्ड मॉडरेशन
1. **कम्युनिटी प्रकार:** “Q&A फ़ोरम”, “डेली डिस्कशन”, “स्पॉट‑फ़ीडबैक थ्रेड” – सभी एक ही `/api/comments` एंडपॉइंट पर स्टोर। |
2. **रोल्स:**
   * **Owner (आपका ID)** – सभी अधिकार, रोल सेटिंग।
   * **Admin** – टिप्पणी/पोस्ट अनुमोदन, हटाना, बैन।
   * **Moderator (स्वेच्छा)** – केवल रिपोर्ट‑हैंडलिंग।
   * **User** – पोस्ट लिख सकता है, लेकिन **प्रकाशन को Admin के Approval** की आवश्यकता (यदि “उच्च‑सुरक्षा” मोड चुनी गई)। |
3. **खर्च‑प्रभावी सेट‑अप:**
   * **Cloudflare Workers + D1** पर सभी कमेंट डेटा रखें (सत्र‑दर्ज़ा)।
   * **सुरक्षित फ़िल्टर:** Regex‑आधारित स्पैम‑ब्लॉक, लिंक‑क्वारन्टीन, गाली‑फ़िल्टर।
   * **समीक्षा फ़्लो:** जब उपयोगकर्ता पोस्ट करता है → `status: pending` → Admin UI (एक साधारण HTML‑डैशबोर्ड) → Approve/Reject। यदि आप “सेट‑अप आसान और कम लागत” नहीं चाहते तो **ऑटो‑अप्रोवल** (सभी पोस्ट तुरंत प्रकाशित) चुन सकते हैं। |

## 7. क्या उपयोगी है / क्या नहीं उपयोगी (Useful vs. Not‑Useful)
| फ़ीचर | उपयोगी? | टिप्पणी |
|---|---|---|
| दैनिक उपयोग‑ट्रैकिंग + ऑटो‑स्क्रीन‑ऑफ़ | ✅ उपयोगी – प्रेरणा, बर्न‑आउट रोकता है। | क्लाइंट‑साइड में रखिए, सर्वर लोड नहीं। |
| मल्टी‑डिवाइस‑लॉग‑इन 3‑डिवाइस सीमा | ✅ उपयोगी – अकाउंट‑शेयर रोकता है, फ़्रॉड घटाता है। |
| Google Analytics 4 + Cloudflare Analytics | ✅ उपयोगी – फ्री, लो‑बैंडविड्थ। |
| Meta Pixel + Audience Network | ✅ उपयोगी – विज्ञापन रिटार्गेटिंग। परंतु विज्ञापन में यदि लागत बढ़े तो हटाएँ। |
| 30‑सेक‑वीडियो‑एड → 5 घंटे ad‑free | ✅ उपयोगी – फ्री‑टियर में रिवॉर्ड‑आधारित मॉडल। |
| Watch‑Ad‑Earn Monetisation (₹1 प्रति रेफ़रल) | ✅ उपयोगी, परन्तु **फ़्रॉड‑रोकथाम** अनिवार्य। |
| रेफ़रल‑कोड को कई डिवाइस पर शेयर करना | ❌ न उपयोगी – फ़्रॉड जोखिम। डिवाइस‑टोकन‑बाउंड बनायें। |
| पूरी‑तरह **Admin‑Approval‑Only** पोस्ट | ⚖️ सीमित उपयोगी – यदि समुदाय छोटा है तो यह कंटेंट डिस्प्ले को धीमा कर सकता है। शुरुआती चरण में **ऑटोपब्लिश** + **स्पैम‑फ़िल्टर** बेहतर। |
| दैनिक स्ट्रीक‑बोनस + PDF‑फ्रीज (₹49) | ✅ बहुत उपयोगी – गहरी जुड़ाव और रेवेन्‍यू दोनों। |
| डैशबोर्ड पर “कुल सेकंड” दिखाना | ❌ अनुशंसित नहीं – छात्रों में तनाव पैदा कर सकता है। केवल “दैनिक मिनट” या “स्ट्रीक‑दिन” दिखाएँ। |

## 8. कार्य‑योजना (Implementation Checklist)
1. **सेशन‑ट्रैकिंग** → क्लाइंट JS (`sessionStorage`) + `/api/usage` Worker राइट।
2. **ऑटो‑स्क्रीन‑ऑफ़** → सेट‑टाइमर + टच‑इवेंट रिवर्स।
3. **डिवाइस‑ID जनरेट** → `localStorage` + FingerprintJS, JWT में एम्बेड।
4. **लॉग‑इन‑सीमा** → D1 टेबल `active_devices` पर क्वेरी, अधिकतम 3।
5. **Google Analytics 4 & Cloudflare Web Analytics** एक साथ इन्टेग्रेट।
6. **Meta Pixel** → हेडर में 1 KB स्क्रिप्ट।
7. **ऑड‑इम्प्लीमेंट** → 30‑सेक‑वीडियो एड (AdSense/Ad Manager) + एड‑फ्री टाइम‑काउंटर।
8. **रिवॉर्ड‑सिस्टम** → D1 `rewards` टेबल, दिन‑प्रति‑इंटरैक्शन अपडेट।
9. **रेफ़रल‑इंजिन** → यूनिक कोड, डिवाइस‑बाउंड, reCAPTCHA‑वेरिफ़िकेशन।
10. **कम्युनिटी‑बैकएंड** → `/api/comments` Worker, स्पैम‑फ़िल्टर, रोल‑बेस्ड UI (Owner/Admin). |

---

**अंतिम सलाह:** प्रारम्भ में **फ्री‑टियर, लो‑कॉस्ट** विकल्पों (Cloudflare Workers, D1, R2, Google Analytics 4, Meta Pixel) का उपयोग करिए। जैसे‑जैसे यूज़र‑बेस बढ़े, एड‑मैनटेनेंस और मॉनेटाइज़ेशन के लिए छोटे‑छोटे एन्हांसमेंट (एड‑फ्री टाइम‑ट्रैकर, रिवॉर्ड‑पॉइंट्स) जोडें। यह सेट‑अप आपको **₹0‑₹20 / माह** में रखेगा, जबकि छात्र को मोटिवेशन, ट्रैकिंग और सुरक्षित भुगतान मिलेंगे।
